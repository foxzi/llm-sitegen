# Examples / Примеры проектов

Папка содержит примеры заданий для генерации сайтов.

## Структура проекта

```
examples/
  project-name/
    content.md   # Контент и бизнес-требования
    technical.md # Дизайн, цвета, стиль (опционально)
    assets/      # Изображения, логотипы (опционально)
```

## content.md - Спецификация контента

Содержит:
- Название и описание бизнеса
- Контактные данные (телефон, адрес)
- Услуги/продукты
- Тексты для страниц
- SEO-требования
- Целевая аудитория

## technical.md - Настройки дизайна

Гибкий файл, может содержать любые секции:

### Обязательные
- **Framework** - какой CSS фреймворк использовать

### Опциональные
- **Color Scheme** - пресет из `modules/colors/` или свои цвета
- **Typography** - шрифты из `modules/fonts/` или системные
- **Style** - общий стиль (modern, classic, minimal)
- **Layout** - структура страницы
- **Animations** - анимации
- **Mobile** - требования к мобильной версии
- **SEO** - мета-теги, schema.org

### Свободные секции
- **Notes / Wishes** - любые пожелания
- **Variants to Generate** - какие варианты создать
- Любые другие секции по необходимости

## Использование

1. Скопируй папку проекта как шаблон
2. Заполни content.md своим контентом
3. Настрой technical.md под свои нужды
4. Передай оба файла LLM для генерации

## Ссылки на ресурсы

В technical.md можно ссылаться на готовые пресеты:
- `modules/colors/ocean-trust` - цветовая схема
- `modules/fonts/inter` - шрифт
- `modules/layouts/landing-page` - шаблон страницы
- `modules/components/cards` - компонент

## Выходные ассеты

В итоговом сайте все ассеты должны лежать в `assets/`:

- `assets/css/`
- `assets/js/`
- `assets/images/`
- `assets/icons/`
- `assets/fonts/`
